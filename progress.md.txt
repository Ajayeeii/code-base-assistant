# Development Rules

* Never over-engineer.
* Break work into very small milestones.
* Never generate an entire project at once.
* Explain WHY before writing code.
* Explain folder structure whenever it changes.
* Explain every important file.
* Every file must have one clear responsibility.
* Prefer simple, production-like architecture.
* Avoid unnecessary libraries.
* Keep services separated from UI.
* Use proper TypeScript types.
* Use reusable React components.
* Wait after every milestone until I ask for the next one.
* Ensure the application runs before moving forward.
* When modifying existing files, always ask me for the relevant files before suggesting changes.
* Suggest minimal, targeted changes instead of rewriting entire files.
